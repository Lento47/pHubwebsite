¿Qué es PymeHub?

PymeHub es una plataforma operativa todo-en-uno diseñada para PyMEs. Centraliza conversaciones, CRM, tareas, documentos y facturación en un solo ecosistema inteligente.

Propuesta de Valor

"Tu socio inteligente que te dice exactamente qué ajustar en tu negocio, basado en datos."

Multi-tenant

Cada empresa tiene su propio espacio de trabajo (workspace) completamente aislado.

Inteligencia Artificial

Analizamos tus datos mes a mes para generar Insights accionables en español.

Inicio Rápido

1

Crear tu cuenta

Regístrate en pymeshub.lat. Automáticamente serás el Owner de tu nuevo Workspace.

2

Conectar Canales

Vincula tu cuenta de Resend (Email) y Meta Cloud API (WhatsApp).

3

Invitar al Equipo

Agrega colaboradores con roles de Admin, Agente o Lector.

Conceptos Clave

Workspace

Tu espacio aislado. Contiene todos tus contactos, conversaciones y documentos.

Canales

Los medios por donde entra la comunicación (Email, WhatsApp, Forms).

Conversaciones

Hilos de mensajes con estados: Nuevo, Abierto, Pendiente y Resuelto.

Roles

Owners, Admins, Agentes y Lectores con permisos granulares.

DESARROLLADORES

Autenticación API

La API de PymeHub usa JWT (JSON Web Tokens) con un sistema de refresh tokens para sesiones seguras de larga duración.

URL Base

https://api.pymeshub.lat

Formato de Respuesta

Todas las respuestas son JSON con la siguiente estructura estándar:

// 200 OK
{
  "data": { ... },
  "message": "OK"
}
// 401 Error
{
  "statusCode": 401,
  "message": "Token inválido",
  "error": "Unauthorized"
}

Endpoints Principales

POST /api/auth/register

Crea una nueva cuenta y workspace de forma programática.

REQUEST BODY: 
{
  "email": "tu@empresa.com",
  "password": "contraseña-segura",
  "name": "Tu Nombre",
  "workspace_name": "Mi Empresa S.A."
}
POST /api/auth/login

Obtiene tokens de acceso iniciales.

Cómo usar el token

Incluye el access_token en el header de todas las peticiones:

Authorization: Bearer {token}

Rotación de Refresh Tokens

Cada vez que renuevas el token, PymeHub emite un nuevo refresh token y el anterior queda inválido automáticamente. Esto protege tu cuenta contra robo de sesiones.

Guía: Asignar WhatsApp a Ventas Automáticamente

Optimiza la respuesta inicial de tu equipo configurando reglas inteligentes que asignen conversaciones según palabras clave o etiquetas.

Lo que necesitas antes de empezar:

  • Cuenta activa en PymeHub (Plan Starter o superior).

  • Al menos dos usuarios con el rol de Agente o Vendedor.

Pasos Detallados

1.

Entrar a Automatizaciones

Ve al menú lateral y haz clic en el icono de rayo (). Dale al botón "Nueva Regla".

2.

Configurar el Disparador (Trigger)

Selecciona "Nuevo Mensaje Entrante". En los filtros, elige "Canal: WhatsApp".

3.

Definir la Acción

Elige la acción "Asignar a Usuario o Equipo". Selecciona a tu equipo de "Ventas" o usa el método "Round Robin" (uno para cada uno).

Tip Pro: Si añades una condición de "Horario Laboral", puedes configurar que fuera de oficina se envíe un mensaje automático de "Te atenderemos mañana".

Preguntas Frecuentes (FAQ)

¿Puedo usar mi número actual de WhatsApp?

Sí, PymeHub funciona con cualquier número de WhatsApp mediante escaneo de código QR (Multi-device). No necesitas cambiar de SIM ni de número.

¿Mis clientes verán que uso PymeHub?

No, la comunicación es transparente. El cliente sigue recibiendo mensajes normales de WhatsApp o Email, mientras tú gestionas todo desde nuestro Inbox profesional.

¿Cómo funciona la IA en el Inbox?

La IA analiza el tono del cliente y te sugiere respuestas rápidas basadas en tu historial. También genera resúmenes diarios de los temas más hablados.

Webhooks

PymeHub puede recibir webhooks de servicios externos para procesar eventos en tiempo real. Actualmente soporta webhooks de Resend (email entrante) y Meta Cloud API (WhatsApp).

Webhook de Email (Resend)

Recibe emails entrantes de clientes y los convierte en conversaciones.

POST https://api.pymeshub.lat/inbound/email/webhook

Headers requeridos

Header Descripción
svix-idID único del evento Resend
svix-signatureFirma HMAC SHA256
X-Workspace-IdID del workspace destino

Payload de ejemplo

{
  "type": "email.inbound",
  "data": {
    "from": "cliente@suempresa.com",
    "subject": "Consulta sobre factura #123",
    "text": "Buenos días, tengo una pregunta sobre..."
  }
}

Webhook de WhatsApp (Meta)

POST https://api.pymeshub.lat/whatsapp/webhook

Verificación (GET)

Meta verifica el webhook con una solicitud GET inicial:

?hub.mode=subscribe&hub.verify_token=secreto&hub.challenge=12345

Eventos soportados

  • messages Mensaje recibido de un usuario
  • message_reads Confirmación de lectura

Reintentos e Idempotencia

PymeHub es idempotente: si recibes el mismo evento dos veces, no se crearán duplicados. Para email se usa el svix-id y para WhatsApp el wamid del mensaje de Meta.

Patrón de Paginación Global

Todos los listados devuelven un objeto meta para facilitar el control de páginas en el frontend:

{
  "data": [...],
  "meta": {
    "page": 2,
    "limit": 25,
    "total": 148,
    "total_pages": 6
  }
}
